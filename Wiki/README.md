# Второй мозг: генная инженерия

Личная база знаний по генной инженерии и смежной молекулярной биологии.
Ведётся в стиле Карпати: разбираемся от первых принципов, пишем, чтобы понять,
а не чтобы заархивировать.

## Принципы (как ведём)

1. **Plain markdown, без инструментов-замков.** Любой редактор, любой git.
   Файлы переживут любую программу.
2. **Атомарность.** Одна заметка — одна идея. Имя файла = суть (`kebab-case`).
   Если в заметке два разных понятия — это два файла.
3. **Плотная перелинковка.** Связываем через `[[имя-файла]]` (без `.md`).
   Ценность базы — в рёбрах графа, а не в узлах.
4. **От первых принципов.** Сначала «что это и зачем», потом механизм по шагам.
   Объясняем так, будто учим другого — это и есть проверка понимания.
5. **Честный фронт незнания.** В каждой заметке блок `## Открытые вопросы` —
   то, что пока не понимаю. Это очередь на изучение, а не стыд.
6. **Источники рядом с утверждением**, а не в конце. Заметка без источника —
   это гипотеза, помечаем её как таковую.
